Git Bash Commands

If you just oppened "Git Bash" App, you'll need to go where you first initialized your Git repository, i.e.: "C:\Users\user\Documents\python-folder".
Else just right click on you working folder (i.e.: python-folder) and select "Git Bash Here".

  • git config --global user.name 'name'
  • git config --global user.email 'email'
  • Add your name & email to Git.
  • git init
  • Initialize a git repository in the folder you are.
  • touch file.extension
  • Create a new file.
  • git status
  • Shows you the status (obviously!).
  • git add .
  • Adds everything (every file).
  • git add file.extension
  • Add a specific file.
  • git rm --cached file.extension
  • Remove a specific file from staging area.
  • git rm --cached -r .
  • Remove everything from staging area.
  • git add *.extension
  • Add all files of that extension.
  • git commit
  • Opens editor (for me notepad++) to, comment changes before committing them.
  • git commit -m 'comment'
  • How to comment changes before committing them, without opening the editor.
  • git log
  • Shows you the log of every commit you've made.
  • git log --graph
  • Draw a text-based graphical representation of the commit history on the left hand side of the output.
  • git reset --hard hash_of_the_commit
  • return to a previous commit:
    (Caution!! You'll lose all other commits after the one you're returning)
  • git clean -f -d (or u can use: git clean -fdx)
  • remove untracked files and directories from the working tree: (check git docs for info)
  • touch .gitignore
  • Create a file so that Git ignores in it's status, whatever other file or folder you put in it:
    (ignore a file file.extension, ignore all files of that extension *.extension, ignore folder /dir...)
  • git push
  • After creating a remote repository in GitHub and following the instructions for git remote and origin ... and
    git push -u origin master, now every time I want to push new changes to GitHub, I use this command.
  • git pull origin name_of_the_branch
  • If you want to pull the changes of a specific branch from a remote repository in GitHub.
    (Caution!! 'pull' does fetch & merge changes on your local repository. It replaces your local repository with what's in the
    remote repository, i.e.: If you use 'git pull origin master' it will replace your local master in your local repository.
    There are also cases that you might get a merge conflict that you'll need to manually resolve: Check google for more details)

  • git stash / git stash -u
  • stashes away all the non-commited changes on your working tree. | use -u to also stash all untracked non-commited files.
  • git stash list | git stash apply index_of_the_stash
  • a list of all the stashes. | brings back all the stash changes of the indexed stash (i.e. 0) to your working tree.
  • git stash show index of the stash / git stash show -p index of the stash
  • shows the file changes of the indexed stashed. | shows you the whole content of the changed files.
  • git stash push -m 'comment' / git stash push -u -m 'comment'
  • Same as git stash or git stash -u but with a comment.
  • git stash branch name_of_the_branch index_of_the_stash
  • Create a new branch from a stash. (brings all the stashed changes in the new branch)
  • git stash pop / git stash pop index_of_the_stash
  • Brings the latest | specified indexed stash to our working tree AND removes the stash from the list.
  • git stash clear
  • Removes all the stashes from the list.
  • git branch name_of_the_branch
  • Create a new branch so you can work and commit changes, without "saving" them to the master branch: (usefull when working with a team)
  • git branch -a / git branch -l / git branch -a | grep search_value
  • List all remote and local branches. | List branch names. | List of specific branch name.
  • git branch -m new_name_of_the_branch
  • Rename branch:
    (entering the command while we are inside the branch we want to rename - to be sure you can 'git checkout name_of_the_branch')
  • git checkout name_of_the_branch
  • Switch between branches.
  • git checkout -b name_of_the_branch
  • Create a new branch and immediately Switch to it.
  • git merge name_of_the_branch
  • Merge another branch to master:
    (entering the command while we are at master branch - to be sure you can 'git checkout master')
  • git merge master
  • Merge master branch to another branch we are currently working on:
    (entering the command while we are at our other branch that we are currently working on - to be sure you can 'git checkout name_of_the_branch')
  • git merge ... --no-ff
  • Creates a merge commit even when a fast-forward would be possible: (helps with better visualising log)
  • git merge --abort
  • In case of i.e. a conflict that git can't auto resolve, you can abort the merge.
