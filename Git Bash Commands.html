Git Bash Commands

If you just oppened "Git Bash" App, you'll need to go where you first initialized your Git repository, i.e.: "C:\Users\user\Documents\python-folder".
Else just right click on you working folder (i.e.: python-folder) and select "Git Bash Here".

  • Add your name & email to Git
  • git config --global user.name 'name'
  • git config --global user.email 'email'
  • Adds everything (every file):
  • git add .
  • Add a specific file:
  • git add file.extension
  • Remove a specific file:
  • git rm --cached file.extension
  • Add all files of that extension:
  • git add *.extension
  • Shows you the status (obviously!):
  • git status
  • Opens editor (for me notepad++) to, comment changes before committing them:
  • git commit
  • How to comment changes before committing them, without opening the editor:
  • git commit -m 'comment'
  • Create a new file:
  • touch file.extension
  • Create a file so that Git ignores in it's status, whatever other file or folder you put in it:
    (ignore a file file.extension, ignore all files of that extension *.extension, ignore folder /dir...)
  • touch .gitignore

  • Create a new branch so you can work and commit changes, without "saving" them to the master branch:
    (usefull when working with a team)
  • git branch name_of_the_branch
  • Switch between branches:
  • git checkout name_of_the_branch
  • Create a new branch and immediately Switch to it:
  • git checkout -b name_of_the_branch
  • Merge another branch to master:
    (entering the command while we are at master branch - to be sure you can 'git checkout master')
  • git merge name_of_the_branch
  • Merge master branch to another branch we are currently working on:
    (entering the command while we are at our other branch that we are currently working on - to be sure you can 'git checkout name_of_the_branch')
  • git merge master
  • After creating a remote repository in GitHub and following the instructions for git remote and origin ... and
    git push -u origin master, now every time I want to push new changes to GitHub:
  • git push
  • If you want to pull the changes of a specific branch from a remote repository in GitHub:
    (Caution!! 'pull' does fetch & merge changes on your local repository. It replaces your
    local repository with what's in the remote repository, i.e.: If you use 'git pull origin master' it will replace your local master in your local repository.
    There are also cases that you might get a merge conflict that you'll need to manually resolve. Check google for more details.)
  • git pull origin name_of_the_branch
