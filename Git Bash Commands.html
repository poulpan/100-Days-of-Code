Git Bash Commands

  • Add your name & email to Git
  • git config --global user.name 'name'
  • git config --global user.email 'email'
  • Adds everything (every file):
  • git add .
  • Add a specific file:
  • git add file.extension
  • Remove a specific file:
  • git rm --cached file.extension
  • Add all files of that extension:
  • git add *.extension
  • Shows you the status (obviously!):
  • git status
  • Opens editor (for me notepad++) to, comment changes before committing them:
  • git commit
  • How to comment changes before committing them, without opening the editor:
  • git commit -m 'comment'

  • Create a new file:
  • touch file.extension
  • Create a file so that Git ignores in it's status, whatever other file or folder you put in it:
  • (Notes for .gitignore: ignore a file file.extension, ignore all files of that extension *.extension, ignore folder /dir...)
  • touch .gitignore
  • Change branch so you can work and commit changes, without "saving" them to the master branch:
    (usefull when working with a team)
  • git branch name_of_the_branch
  • Switch between branches:
  • git checkout name_of_the_branch
  • Merge another branch to master:
    (entering the command while we are at master branch)
  • git merge name_of_the_branch
  • After creating a repository in GitHub and following the instructions for git remote and origin ... and
    git push -u origin master, now every time I want to push new changes to GitHub:
  • git push
